# v1.1.2 リリースノート

## 概要
- 対象: iRIC_DataScope
- リリース日: 2026/01/16
- 目的: 左右岸水位抽出（lr_wse）の入力判定強化と出力整理

## ユーザ向けの主な変更
- 左右岸水位抽出に「最小水深」入力を追加。`depth(m)` が **入力値以下** の場合、その岸の `watersurfaceelevation(m)` / `elevation(m)` を **"取得不可"** として出力。
- `depth(m)` 列が存在しないデータで「最小水深」を使うとエラーになります。
- `t=0` の時系列は抽出対象から除外されます。
- 左右岸水位抽出の Summary 列順を整理（`t, I, J, X, Y, ...` の順）。
- 処理完了後に左右岸水位抽出の GUI が自動で閉じないようになりました。
- KP 列が `97.6k` のような末尾 `k/K` 付きでも設定CSVを読み込めるようになりました。
- Summary 出力時に KP が数値の場合、末尾に `k` を付与して出力します。

## ドキュメント
- 左右岸水位抽出の仕様に `t=0` 除外を追記。
- 左右岸水位抽出の最小水深仕様メモを追加。
- ユーザ向けドキュメントの整理（GUI説明の拡充）。

## 影響範囲
- 影響を受ける機能: 左右岸水位抽出（lr_wse）
- 互換性: 既存の出力形式は概ね維持されますが、列順と「取得不可」表記が追加されます。
